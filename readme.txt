CI/CD es una forma de automatizar un repositorio para el desarrollo de un código. 
Un repositorio con integración constante (CI) se encargará de controlar que una nueva versión del código pueda mezclarse con las versiones que ya están almacenadas en el mismo, evitando conflictos. En caso de no haber problemas, el código se subirá de manera automática.
Un repositorio con entrega y distribución continuas (CD) se va a encargar de almacenar los cambios de un código constantemente y desplegarlo directamente en el área de producción o en algún área de testeo en caso de que así haya sido establecido anteriormente por los desarrolladores.

Beneficios de usar CI/CD:
La integración constante ayuda a evitar que se generen conflictos en casos donde se desarrolla más de una división de algún código al mismo tiempo.
La entrega y distribución continua permite que el código nuevo se implemente con el minimo esfuerzo posible. Es una solución al problema de la falta de supervisión y comunicación entre los equipos comerciales y de desarrollo


Ambientes de Trabajo:
El desarrollo de un código puede llevarse a cabo en distintos tipos de ambientes para asegurar que tenga una buena presentación final. Los cuatro ambientes principales que se pueden identificar y diferenciar entre sí son Desarrollo, Pruebas (o testing), preproducción (o staging) y producción. 
1 - Ambiente de desarrollo: Es donde se crea el código y se lo convierte en un ejecutable. En este ambiente el equipo de programación se encarga de integrar al código todas sus bases y funcionalidades principales.
2 - Ambiente de pruebas (Testing): A diferencia del ambiente anterior aquí se maneja código o versiones estables del programa, al que se le realizan pruebas de seguridad y funcionalidad por parte del equipo de pruebas. Aun así, este ambiente mantiene las características de configuración y recursos del ambiente anterior para poder realizar cambios y mejoras.
3 – Ambiente de preproducción (Staging): Este ambiente pretende ser lo más parecido posible al Ambiente final de producción. Se ubica al programa aquí para realizar pruebas avanzadas que aseguren que este se presente de la manera esperada en su entrega final.
4 - Ambiente de producción: Finalmente el programa es lanzado a operar para los usuarios finales. Aun se siguen realizando ajustes por parte del equipo de desarrollo para mantener disponible al servicio.
